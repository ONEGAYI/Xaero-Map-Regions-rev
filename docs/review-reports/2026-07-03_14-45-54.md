## 审查结论

本次对 Xaero Map Regions Rev 仓库全量源代码（40 个 Java 主文件，约 2832 行）进行了高强度召回模式审查，执行了 9 个独立审查角度、对候选发现逐项验证、并完成遗漏扫描。

**确认的正确性 bug 共 13 项（P0/P1/P2）**，其中最严重的 7 项 P0 涵盖：投影坐标系分裂、SavedData 静默数据丢失、凹多边形渲染错误、网络线程阻塞、CJK 名称发包失败、createPoint 误报成功、编辑会话被意外清空。

**核心风险主题**：
1. **数据安全**：SavedData 静默覆盖 + createpoly 绕过校验 + readRegion 无点数上限 —— 三处独立路径都可能导致区域数据丢失或不可恢复。
2. **网络层健壮性**：decode 在网络线程做重计算 + readUtf 无限制 + handleRefresh 无权限/无限流 —— 存在客户端 DoS 与 OOM 风险。
3. **客户端交互一致性**：双投影实例 + 菜单 fall-through + 子屏 reset + 硬编码按键 —— 编辑器交互在多个边界下行为异常。
4. **国际化**：CJK 文本因 char/字节度量不匹配在发包时崩溃。

建议优先处理 P0 数据安全与正确性问题，再处理 P1 交互与安全问题。
