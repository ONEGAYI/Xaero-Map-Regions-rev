# 重叠区域悬停气泡与标签避让设计

> 日期：2026-07-12
> 状态：待实现

## 背景与问题

Xaero 世界地图覆盖层当前逐区域完成填充、边界、标签与悬停气泡渲染。当鼠标位于多个区域的重合部分时，每个命中区域都会在同一坐标调用一次 `renderTooltip`，导致多个气泡互相覆盖。同时，各区域的行内标签只检查文字是否能放入自身多边形，没有跨区域碰撞检测，因此相邻或重叠区域的标签可能叠字。

本次修复同时覆盖 NeoForge 1.21.1 主力维护线与 Forge 1.20.1 历史维护线，并保持两端显示语义一致。

## 目标

1. 同一帧最多渲染一个区域悬停气泡。
2. 气泡按行列出鼠标位置命中的区域，并对大量重叠区域设置明确边界。
3. 行内标签发生屏幕空间碰撞时，只保留高优先级标签。
4. 显示优先级与现有“顶层 → 底层”循环选中语义一致。
5. 将可判定的布局规则封装为不依赖 Minecraft 渲染 API 的纯逻辑，并以契约测试覆盖。

## 非目标

- 不改变区域填充、边界和选中轮廓的绘制顺序。
- 不改变区域点击命中或循环选中状态机。
- 不增加可滚动气泡、标签拖拽、标签配置项或复杂自动排版。
- 不解决 Xaero 自身元素与本模组标签之间的碰撞。

## 根因

`XaeroMapOverlayRenderer` 当前在可见区域循环中立即调用 `XaeroMapOverlayController.renderRegionDecorations(...)`。控制器随后针对单个区域立即判断悬停、渲染气泡并绘制标签。由于每次调用只能看到当前区域，既无法把多区域悬停结果合并，也无法知道其他标签已经占用的屏幕矩形。

根因是需要全局信息的布局决策被放在了逐区域即时绘制阶段，而不是单帧候选收集后的统一布局阶段。

## 方案比较

### 方案 A：统一收集并布局（采用）

先收集当前帧全部可见区域的投影和显示候选，再统一完成标签碰撞过滤与悬停气泡生成，最后绘制。该方案能从根因上消除重复气泡，并使标签避让结果稳定、可测试。

### 方案 B：逐区域绘制时维护静态帧状态

在控制器中记录“本帧是否已经显示气泡”和已占用标签矩形。改动表面较小，但需要可靠识别帧边界，容易产生跨帧残留和调用顺序耦合，不采用。

### 方案 C：仅显示最后命中的区域

可以避免气泡叠加，但会隐藏重合区的其他区域信息，也不能解决行内标签碰撞，不采用。

## 总体架构

### 帧级候选收集

平台侧 `XaeroMapOverlayRenderer` 在单次地图渲染中：

1. 过滤维度不符、点数不足或屏幕外的区域。
2. 计算每个可见区域的投影点。
3. 按现有顺序绘制区域填充、边界和选中轮廓。
4. 为每个可见区域计算行内标签候选和鼠标悬停状态，但不立即绘制标签或气泡。
5. 将整帧候选交给统一布局逻辑。
6. 绘制通过碰撞过滤的标签。
7. 根据全部命中区域渲染至多一个合并气泡。
8. 最后绘制编辑器工具栏、选中区域 HUD 和上下文菜单，保持现有覆盖层次。

NeoForge 与 Forge 保持对应实现；纯布局数据结构与规则应优先放入 `common` 模块，避免两份平台代码分别演化。

### 显示优先级

候选优先级从高到低为：

1. 当前已选中且位于候选集合中的区域。
2. 其余区域按现有渲染顺序的逆序，即顶层到最底层。

同一优先级不存在随机排序；输入列表与区域 ID 共同保证结果可重复。该顺序同时用于悬停气泡和标签碰撞过滤。

## 合并悬停气泡

### 内容

鼠标未命中任何区域时不显示区域气泡。命中一个或多个区域时，仅调用一次 Minecraft `renderTooltip`，每个显示区域占一行，文本使用 `Region.label()`。

每个区域明细行的文字颜色跟随该区域的 `labelColor`，但忽略透明度参数。平台层仅提取颜色值的 RGB 低 24 位（等价于 `labelColor & 0x00FFFFFF`）并写入对应 `Component` 样式，确保半透明或全透明标签色不会导致气泡文字难以辨认。省略汇总行不对应具体区域，保持 Minecraft 默认提示文字颜色。

当前选中区域若在命中集合中则置于首行；其他区域按顶层到最底层排列。

### 大量重叠区域边界

可显示区域行数同时受自适应上限和硬上限约束：

- 硬上限为 8 个区域。
- 自适应上限根据屏幕高度、字体行高和气泡上下安全边距计算，确保气泡内容不会超过可用垂直空间。
- 实际区域行数取命中数量、自适应上限与硬上限三者的最小值。
- 当存在未显示区域时，预留最后一行显示 `……还有 N 个区域`；因此区域明细行数需相应减少一行。
- 在极小窗口中，至少保留一行有效内容：优先显示最高优先级区域；若仍有遗漏，则在空间允许时显示汇总行。

省略汇总文字通过中英文语言键提供，不在平台代码中硬编码。

## 行内标签碰撞避让

每个现有 `InlineLabel` 候选扩展出屏幕占用矩形。矩形至少覆盖文字实际宽高，并在四周加入小幅安全间距，避免阴影或紧贴边缘造成视觉粘连。

布局器按显示优先级遍历候选：

1. 候选与所有已接受矩形均不相交时，接受并占位。
2. 发生相交时，隐藏当前低优先级标签。
3. 不移动标签锚点，不尝试上下左右搜索新位置，避免标签从所属区域中心漂移。

边界接触是否视为碰撞必须在纯逻辑中明确定义；本设计将包含安全间距后的矩形接触视为碰撞，从而保留清晰的视觉间隔。

隐藏行内标签不会影响悬停气泡。用户仍可在重合区通过合并气泡查看所有区域名称，并通过现有循环点击选中具体区域。

## 组件职责

- `XaeroMapOverlayRenderer`：负责单帧可见区域投影、候选汇总及最终绘制顺序。
- `XaeroMapOverlayController`：保留单区域边界、选中轮廓、标签候选计算和平台绘制适配；不再针对单个区域立即渲染悬停气泡。
- 公共布局模型：表达带区域 ID、标签文字、优先级和矩形的候选，完成稳定排序、标签碰撞过滤和气泡行数裁剪。
- `RegionLabelDisplay`：继续负责单个多边形内部的标签尺寸适配；平台重复逻辑若无法直接移入 common，则两端保持同构并由公共布局规则约束。

具体类名可在实施计划中根据现有包职责确定，但不得把 Minecraft GUI 类型引入 `common`。

## 边界与异常处理

- 空标签候选、退化多边形和屏幕外标签沿用现有隐藏行为。
- 字体宽度或行高由平台层测量后以整数数据传入纯布局逻辑。
- 重复区域 ID 不应由正常缓存产生；布局器按输入候选处理，不额外吞掉合法的同名不同 ID 区域。
- 多个区域可以拥有相同标签，气泡仍逐区域显示，不按文本去重。
- 区域同步在帧间变化时，每帧重新收集候选，不保存跨帧占位状态。

## 测试策略

遵循 TDD，先增加失败契约测试，再实现修复。

### 公共纯 Java 测试

- 单个命中区域生成一行气泡。
- 多个命中区域按“已选中优先、其余顶层到底层”排序。
- 每个区域气泡行使用对应标签色的 RGB 分量，并忽略 alpha 分量。
- 省略汇总行保持默认提示文字颜色。
- 命中数量小于上限时全部显示。
- 超过硬上限时最多显示规定行数，并正确计算 `N`。
- 自适应上限小于硬上限时按屏幕高度裁剪。
- 极小可用高度仍保留最高优先级区域。
- 不相交标签全部保留。
- 相交标签只保留高优先级候选。
- 当前选中标签在碰撞时覆盖普通顶层标签。
- 矩形安全间距导致的边界接触按碰撞处理。
- 输入顺序固定时输出顺序稳定。

### 平台测试

- NeoForge 与 Forge 的渲染编排只生成一次合并气泡调用所需的数据。
- 标签候选与公共布局结果正确转换为各版本 `GuiGraphics` 绘制调用。
- 现有 `RegionLabelDisplayTest` 继续通过，确保单区域内部适配未回归。

## 验收标准

1. 鼠标位于任意多区域重合区时，屏幕上只有一个本模组区域悬停气泡。
2. 气泡能展示多个区域名称，各区域行文字颜色跟随标签色但忽略透明度；超过容量时以默认文字色的 `……还有 N 个区域` 收束，且不会无限增高。
3. 行内区域标签不再彼此叠字；碰撞时显示当前选中区域或更高层区域的标签。
4. 离开重合区后，未碰撞区域的标签恢复正常显示。
5. NeoForge 1.21.1 与 Forge 1.20.1 行为一致，现有区域选择、编辑和 HUD 功能无回归。
