# Xaero 地图区域标注与编辑设计

## 背景

当前 MVP 已能完成服务端区域持久化、登录同步、客户端缓存，并在 Xaero World Map 上渲染服务端同步的区域填充。下一阶段需要把体验推进到可用产品：区域应能显示地名标签，且管理员能直接在 Xaero 地图中创建和删除区域。

参考模组的公开说明给出的核心交互是：在 Xaero 地图中按 `R` 进入区域绘制，绘制时用鼠标中键放置多边形点；区域管理快捷键也限定在 Xaero 地图场景内响应，普通游戏视角不触发本模组快捷键。本文按干净实现处理这些交互，不复制或反编译目标模组。

## 范围

### 本阶段包含

- 区域标注字段：
  - 地名标签，默认等于区域名称。
  - 标签字体颜色，可使用 ARGB/RGB 十六进制输入。
- 区域渲染：
  - 保留现有多边形填充。
  - 在区域几何中心渲染单行地名标签。
  - 标签颜色使用服务端同步的标签颜色。
- 地图内编辑模式：
  - Xaero 地图页增加 `Edit` / `Done` 切换按钮。
  - 注册可在 Minecraft 控制设置中修改的 Forge `KeyMapping`，默认 `R` 在 Xaero 地图中切换绘制模式。
  - 注册可修改的区域管理 `KeyMapping`，默认 `T` 仅在 Xaero 地图页打开区域管理器。
  - 鼠标中键在当前地图光标位置添加多边形顶点。
  - `Enter` 保存当前多边形，`Esc` 取消当前绘制。
  - 编辑模式下选中区域后，右键菜单提供删除、改区域颜色、改标签内容、改标签颜色。
- 区域输入：
  - 保存新区域前弹出简单输入界面，填写区域名称、标签文字、区域颜色、标签颜色。
  - 默认名称和标签可预填为 `region_<timestamp>`，颜色预填为当前 MVP 默认色。
- 服务端权威：
  - 客户端编辑只发送请求包。
  - 服务端校验 OP 且创造模式权限。
  - 服务端写入 `SavedData` 后广播完整区域快照。
- 简单区域选择：
  - 编辑模式下鼠标左键选择光标所在区域。
  - 选中区域后地图上显示更明显的边框。
  - 选中区域后右键打开上下文菜单。
- 验证：
  - 单元测试覆盖新增字段的 NBT 和网络往返、屏幕识别、编辑状态机的纯逻辑。
  - GUI 烟测覆盖创建区域、显示标签、删除区域、重新登录后同步。

### 本阶段暂不包含

- 拖动或编辑已有多边形顶点。
- 区域列表搜索、排序和批量管理。
- 标签避让、缩放自适应、描边样式高级配置。
- 严格读取 Xaero 内部缩放和平移状态来做到像素级投影一致。
- 上传自定义图标。
- 进入区域标题或 Traveler's Titles 集成。

这些能力会在本阶段稳定后单独设计。

## 数据模型

`Region` 增加两个字段：

- `label`：地图显示文本。不能为空，创建时默认等于 `name`。
- `labelColor`：标签字体颜色。使用现有 `ArgbColor` 类型。

为了保持兼容：

- NBT 读取旧数据时，如果缺少 `label`，使用 `name`。
- NBT 读取旧数据时，如果缺少 `labelColor`，使用白色 `0xFFFFFFFF`。
- 网络包在同一开发版本内同步更新字段；客户端和服务端协议版本保持一致即可。

## 服务端接口

新增区域编辑请求包：

- `CreateRegionRequestPacket`
  - `name`
  - `label`
  - `fillColor`
  - `labelColor`
  - `points`
- `DeleteRegionRequestPacket`
  - `regionId`
- `UpdateRegionStyleRequestPacket`
  - `regionId`
  - `fillColor`
  - `label`
  - `labelColor`

处理规则：

1. 服务端收到请求后取发送玩家作为权限主体。
2. 权限必须满足 OP 且创造模式。
3. 创建区域时服务端使用玩家当前维度作为最终维度，请求包不携带客户端维度字段。
4. 区域名称生成 `RegionId`，重复 id 时拒绝并给玩家返回聊天错误。
5. 点数少于 3、非法多边形、非法颜色时拒绝。
6. 删除和改色/改标签时只允许操作当前服务端已存在的区域。
7. 成功后写入 `SavedData`，广播完整快照。

现有 `/region createpoly` 保留为调试和管理命令，并扩展可选参数或新增命令以支持标签与标签颜色。

## 客户端编辑状态

新增客户端编辑状态对象，职责仅限本地交互：

- 是否处于编辑模式。
- 是否处于绘制模式。
- 当前临时点列表。
- 当前选中区域 id。
- 最近一次鼠标所在世界坐标。

状态转换：

1. 地图页点击 `Edit` 或按 `R`，进入绘制模式。
2. 鼠标中键添加顶点。
3. 顶点数达到 3 后，`Enter` 打开保存输入界面。
4. 输入确认后发送 `CreateRegionRequestPacket`。
5. `Esc` 清空临时点并退出绘制模式。
6. 编辑模式下左键选区。
7. 选中区域后右键打开上下文菜单。
8. 菜单点击删除时发送 `DeleteRegionRequestPacket`。
9. 菜单点击改色、改标签时打开编辑表单，确认后发送 `UpdateRegionStyleRequestPacket`。

临时绘制只在客户端内存中存在，不写入本地持久化。

## 地图坐标转换

本阶段继续将 `MapProjectionAdapter` 作为唯一投影边界：

- 渲染已有区域时使用当前适配器投影。
- 鼠标添加点时先做反向投影，把屏幕坐标转换为世界 `x,z`。
- 反向投影采用与当前正向投影一致的近似比例，保证 MVP 内创建后能在同一地图视图中看到区域。

后续若需要与 Xaero 缩放/拖拽完全一致，再单独研究反射或可用 API 接入真实地图相机参数。

## UI 设计

### 可配置快捷键

本阶段所有键盘入口都必须注册为 Minecraft 控制设置中的可修改按键：

- `xaeroregionsrev.key.toggle_region_drawing`：默认 `R`，仅在 Xaero 地图页生效，用于进入或退出区域绘制。
- `xaeroregionsrev.key.open_region_manager`：默认 `T`，仅在 Xaero 地图页打开区域管理器。

代码中不能直接硬编码检查 `R` 或 `T` 的物理键值；实现应读取 Forge `KeyMapping` 状态，并通过 map-screen-only 的 conflict context 或事件入口确保普通游戏视角不响应本模组快捷键。文档和 GUI 提示可以显示默认键位，但提示文本应尽量从当前 key mapping 取值。

### Xaero 地图覆盖控件

在地图左侧或右上角绘制一个小型按钮：

- 非编辑模式显示 `Edit`。
- 编辑模式显示 `Done`。

按钮只在 Xaero 地图页显示，不出现在普通 HUD。样式保持克制，避免遮挡地图主体。

### 绘制反馈

绘制模式中：

- 已放置顶点以小方块或圆点显示。
- 顶点之间以线段连接。
- 顶点数达到 3 时显示临时半透明填充。
- 当前模式提示使用简短文本，如 `Region Edit: middle-click points, Enter save, Esc cancel`。

### 保存输入界面

简单表单包含：

- 区域名称。
- 显示标签。
- 区域颜色。
- 标签颜色。
- 保存按钮。
- 取消按钮。

输入界面只负责收集文本；所有最终校验仍由服务端执行。

### 选中区域右键菜单

编辑模式下左键选中区域后，鼠标右键打开小型上下文菜单。菜单项：

- `Delete`：删除选中区域，点击后弹出确认框，确认后发送 `DeleteRegionRequestPacket`。
- `Fill Color`：打开颜色输入表单，只修改区域填充颜色。
- `Label Text`：打开文本输入表单，修改地图显示的地名标签内容。
- `Label Color`：打开颜色输入表单，修改标签字体颜色。

菜单只对当前选中区域生效。若没有选中区域，右键不打开菜单或只显示“无选中区域”的短提示。

## 渲染规则

区域渲染顺序：

1. 普通区域填充。
2. 选中区域高亮边框。
3. 区域标签文本。
4. 临时绘制点和线。
5. 编辑模式按钮和提示。

标签位置使用多边形点的平均中心作为 MVP 实现。标签字体颜色使用 `labelColor`，默认白色。

## 错误处理

- 服务端拒绝创建或删除时，向玩家发送聊天错误，不修改数据。
- 客户端请求包解析失败时记录日志并丢弃。
- 保存输入界面发现明显非法颜色时可直接提示，但不能替代服务端校验。
- Xaero 地图未安装或屏幕类名变化时，编辑按钮和地图交互不可用，但服务端命令和同步仍可工作。
- 客户端退出地图时清空临时绘制状态。

## 测试策略

- `RegionTest`：覆盖 `label`、`labelColor` 默认值和必填校验。
- `RegionNbtCodecTest`：覆盖新字段往返和旧 NBT 默认兼容。
- `RegionSyncPacketTest`：覆盖新字段网络往返。
- 新增编辑状态机测试：覆盖进入绘制、添加点、取消、保存前条件、选中与删除请求条件。
- 新增快捷键测试或轻量封装测试：确认默认键位注册为 `R` / `T`，并且交互逻辑读取 key mapping 而不是硬编码键值。
- 新增右键菜单状态测试：覆盖选中区域后可删除、改区域颜色、改标签内容、改标签颜色。
- 新增颜色解析测试：覆盖 `0x`、`#`、RGB 自动补 alpha、非法字符。
- GUI 烟测：
  - `runClient` 进入测试世界。
  - 打开 Xaero 地图，点击 `Edit` 或按 `R`。
  - 用鼠标中键添加至少 3 个点。
  - 保存区域，设置标签颜色。
  - 看到区域填充和标签文本。
  - 在控制设置中能看到并修改本模组的 `R` / `T` 默认快捷键，且普通游戏视角按下不会触发本模组动作。
  - 编辑模式下选中区域，右键菜单能删除、改区域颜色、改标签文字和改标签颜色。
  - 重新进入世界后，区域仍同步显示。
  - 通过右键菜单删除区域后，地图显示消失。

## 验收标准

- 构建通过：`.\gradlew.bat test build`。
- 服务端创建、删除区域仍由权限校验保护。
- 客户端通过地图内编辑创建区域后，服务端日志显示收到请求并广播同步。
- 客户端日志显示缓存更新。
- Xaero 地图显示区域填充和自定义颜色地名标签。
- `R` 和 `T` 作为默认快捷键注册在 Minecraft 控制设置中，并可被用户重绑；两者只在 Xaero 地图场景响应。
- 编辑模式下选中区域后，右键菜单可以删除区域、修改区域颜色、修改标签内容、修改标签字体颜色。
- 退出并重新进入世界后区域和标签仍存在。
- 地图内删除区域后，服务端持久化删除并同步到客户端，地图上不再显示该区域。
